## ðŸ“‹ Documentation Standards

All documentation in this project follows these standards:

### Markdown Files
- Use clear headings and hierarchical structure
- Include table of contents for docs > 100 lines
- Use code blocks with language specifiers
- Add examples where applicable
- Keep lines under 120 characters when possible

### Code Examples
- Always include language identifier (```typescript, ```sql, etc.)
- Show both correct and incorrect patterns when teaching
- Include comments explaining complex logic
- Provide complete, runnable examples

### Updates
- Update documentation when changing related code
- Include version numbers for breaking changes
- Add dates to changelog entries
- Archive outdated documentation rather than deleting
